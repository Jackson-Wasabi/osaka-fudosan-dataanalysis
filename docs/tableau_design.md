# Tableau ダッシュボード設計仕様（Step 13）

作成: 2026-06-17 / 対象: 中古マンション仕入れ一次スクリーニング ダッシュボード（3画面）
位置づけ: この仕様に沿って Tableau Desktop で組み立てる。データは検証済み（緯度経度・NULL・フラグ整合チェック合格）。

## 0. 設計原則（最重要・額面で買わせない）
- 割安スコアは**単独で見せない**。必ず `high_risk_share`（リスク高率）・`stable_flag`（安定候補）・`building_risk_label` を隣に置く。
- すべての画面に注記：「**調査候補の一次スクリーニング。買い判断ではなく、現物・謄本確認が前提**」。
- 対象は **20-60㎡帯・2025年・大阪市・取引10件以上の66駅**であることを明記。

## 1. データソース（BigQuery コネクタ・OAuth）
プロジェクト: `osaka-fudosan-dataanalysis`
- **A（駅単位・66行）**: `osaka_real_estate_marts.mart_opportunity_list` … マップ・比較表用
- **B（物件単位・2,097行）**: `osaka_real_estate_intermediate.int_property_deviation` … 詳細散布図・リスク内訳用
- **関連付け**: A と B を `station_name` でリレーション（結合でなくリレーションシップ推奨＝粒度の異なる集計のfan-out回避）。
- 画面間の連動は**ダッシュボードアクション（フィルター）**を `station_name` で設定（マップで駅クリック→詳細が絞られる）。

## 2. パラメーター・計算フィールド（共通）

### パラメーター `配点パターン`（文字列: バランス / 割安重視 / リスク重視）
3配点はテーブルに計算済み列があるので、パラメーターで表示する列を切り替える。

```
// [スコア（選択配点）]
CASE [配点パターン]
  WHEN "バランス"   THEN [priority_score]
  WHEN "割安重視"   THEN [score_discount]
  WHEN "リスク重視" THEN [score_risk]
END
```
```
// [順位（選択配点）]
CASE [配点パターン]
  WHEN "バランス"   THEN [priority_rank]
  WHEN "割安重視"   THEN [rank_discount]
  WHEN "リスク重視" THEN [rank_risk]
END
```

### 表示用の計算フィールド
```
// [要確認フラグ]  リスク高率が高い駅を強調（しきい値は調整可）
IF [high_risk_share] >= 0.30 THEN "要確認（旧耐震×改装不明 多）" ELSE "—" END
```
```
// [割安度(%表示)]  under_level は負ほど割安。正負を直感的に
ROUND(-[under_level]*100, 1)   // 正の値ほど割安
```
```
// [実勢中央(万円/㎡)]  ROUND([actual_median_price_per_sqm]/10000, 1)
```
```
// [安定候補ラベル]  IF [stable_flag] THEN "安定候補" ELSE "" END
```

## 3. 画面①：駅マップ（メイン）
- **マーク**: 自動（円）/ 地図。緯度 `station_lat`、経度 `station_lon`。
- **色**: `[スコア（選択配点）]`（連続・割安が高得点＝目立つ色）。または `[割安度(%表示)]`。
- **サイズ**: `n_transactions_2025`（取引の厚み）。
- **強調**: `[要確認フラグ]` を形/枠線で区別（例：要確認駅は枠線を濃く）。`stable_flag` を別マークレイヤーで★表示も可。
- **フィルター**: `配点パターン`（パラメーター）/ `stable_flag`（安定候補のみ表示トグル）。
- **ツールチップ**: 駅名・区・スコア・順位・割安度・`high_risk_share`・取引件数・「調査候補」注記。
- **アクション**: クリックで画面②へ `station_name` フィルター。

## 4. 画面②：駅詳細（drill・選択駅）
画面①で選んだ駅を `station_name` フィルターで受ける。4部品：
1. **スコアカード**（ソースA）: `sc_value/sc_reliability/sc_land/sc_quality/risk_penalty` を横棒で内訳表示＋総合スコア。
2. **3配点の順位**（ソースA）: バランス/割安重視/リスク重視の `rank_*` を並べ「配点で順位がどう動くか」。`stable_flag` を明記。
3. **リスク理由**（ソースB）: その駅の `building_risk_label` を物件数で積み上げ（旧耐震×改装不明＝リスク高 が一目で）。
4. **物件散布図**（ソースB）: x=`predicted_price_per_sqm`、y=`actual_price_per_sqm`、色=`building_risk_label`（または `station_relative_deviation`）。**45度の参照線**を引き、線の下＝予測より安い（割安側）。点のツールチップに面積帯・築年・徒歩・乖離。

## 5. 画面③：候補比較表（66駅ランキング）
- **行**: 駅名（`[順位（選択配点）]` で並べ替え）。
- **列/指標**: `[スコア（選択配点）]`・バランス/割安重視/リスク重視の各順位・`stable_flag`・`high_risk_share`・`n_transactions_2025`・実勢中央。
- **ハイライト**: `stable_flag`=安定候補を色分け。`high_risk_share`をヒートで。
- **フィルター**: `配点パターン`で並べ替えが変わり「順位の入れ替わり」を体感。安定候補のみ表示トグル。
- **狙い**: 「**配点は固定の正解でない。3配点中2回以上Top20＝安定候補は頑健**」を1画面で説明。

## 6. ダッシュボード組み立て・体裁
- 3シートをダッシュボードに配置。①を主、②③をタブまたは下段。
- タイトル：「大阪市 中古マンション（20-60㎡）仕入れ一次スクリーニング ― 2025年・66駅」。
- フッター注記（全画面共通）：「調査候補。買い判断でなく現物・謄本確認前提。割安＝相場より下振れであり好機/衰退はデータ単独で判別不可（要検証）。成約価格情報ベース＝個別物件は売出でない。」
