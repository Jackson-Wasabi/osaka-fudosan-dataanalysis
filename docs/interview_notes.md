# 面接 想定問答（このプロジェクトのアピールポイントと追撃対策）

作成: 2026-06-16 / 用途: 面接前に見返す。技術詳細は decision_log / verification_notes / step10_model_plan を参照。

## 立ち位置（最初に言うこと）
「中古マンションの仕入れ一次スクリーナーを公開データで作りました。**査定（適正価格の算定）ではなく、人が精査すべき候補エリアを絞る道具**です。精度より、**駅内の相対順位の頑健さと、限界を自分で説明できること**を重視しました。」

---

## アピール① 逆選択をデータで仕分けた（平野の例）

**主張**: ランキングを額面で信じず、上位駅が本物の割安か、データに写らない理由で安い逆選択かを、手元データの指紋（旧耐震率・大型偏り・築年・極端外れ値・薄商い）で仕分けた。

**想定追撃**: 「築古が多いだけでは？ モデルは築年数を特徴量に入れてるんだから、古さは既に価格に反映済みのはず。なぜまだ割安に出る？」

**答え（断定を避けるのが肝）**:
- 「“老朽だから安い”とは**断定しません**。Model C は築年数を含むので、古さは既に減価済みのはずです。」
- 「私が示したのは、**割安シグナルが最も信用できないセグメント（平野＝築52年・大型95%・周縁・21件と薄商い）に集中している**という事実。古い周縁・少数取引では駅中央値の信頼性も低く、`割安 vs モデル`は買い場でなく**軟調/流動性低下/隠れ瑕疵**の可能性が高い。」
- 「だからメカニズムを断定せず、**リスク仕分けとして②要注意に降格**しました。」

→ NG例: 「平野は老朽大型だから安い」（モデルが古さを織込済なので論理破綻）。
→ OK: 「割安シグナルが不確実な領域に集中しているので降格した」。

## アピール② 既存の外れ値フラグの穴を発見

**主張**: 極端割安（駅相対 −30〜−70%）の物件が、既存の `outlier_pps_flag`（市全体IQRの絶対判定）を**すり抜けている**。松屋町の −70%物件（実勢21.8万 vs 予測53.7万）が無印で残存。

**想定追撃**: 「なら最初からその相対フラグを作れば良かったのでは？」

**答え**:
- 「既存フラグは staging 段階の**絶対的なデータ誤り検出**（D-002）。一方、**駅に対して相対的な異常は、予測と駅相対乖離を計算した後（Step 11）でないと観測できません**。」
- 「だから自然に次イテレーションの課題で、`駅相対の外れ値フラグ`を加えれば逆選択/データ誤りを自動隔離できる、という具体的改善提案にしました。」

→ 設計の階層（staging=絶対誤り / mart=相対異常）を理解している証明になる。

## アピール③ ビジネスインパクトを数値で

**主張**: 2,097件・151駅 → 上位10駅（309件＝全駅の約7%）に圧縮。割安プール472件（22.5%）を抽出。上位10駅を**本物4／逆選択1／要確認5**に仕分け。

**想定追撃**: 「“本物4駅”は主観では？」

**答え**:
- 「主観ではなく**再現可能な4基準**です——旧耐震率ほぼ0・築年が新しめ・極端外れ値なし・駅内乖離が一様。これを満たすのが堺筋本町/玉造/大国町/森ノ宮の4駅。判定ロジックは verification_notes に明記しています。」

→ 「眼力ではなく基準」を強調。

## モデル選定で語れる芯（深掘りされたら）

- **なぜ Model C？**: Baseline(駅中央値)→C で MAE 18.6万→12.1万に改善。公示価格D・木モデルEを足しても伸びず＝**実測で棄却**（“決め打ちしない”）。
- **集計最良の Model F をなぜ捨てた？**: F は全市一律の値上がり補正で hit10 最良だが、**周縁の割安区を+25〜40%過大予測**し「割安候補」を誤検出。**一様なバイアスは順位を保つが、構造化したバイアスは順位＝道具の核心を壊す。MAEが良い≠スクリーニングに良い**。
- **精度は高いのか？**: 高くない（中央ズレ約18%）。ただし**公開データの粒度（階数・住所・リフォーム状態の欠落）で決まる下限**で、複雑なモデルでも改善しないことを実測。だから査定でなくスクリーナーと位置づけ、**駅内の相対順位**で評価。

## スコア設計で語れる芯

- 100点満点だが、解剖したら**品質15・リスク20がほぼ機能していなかった**（実測のばらつき・相関で確認）。改装94%欠損が品質軸を殺し**偽の66点天井**を作っていたので修正（D-029）。
- リスクは旧耐震単独に広げない＝**価格に織込済で二重計上回避**（D-030）。
- 「**無い差を捏造しない**」——品質は全駅一様なら一様と正直に出す。

## パイプライン監査で自分から言える弱点（誠実さの加点・D-031）

「全工程を自己監査しました。結果を壊すバグはありませんが、正直に言うべき限界が4つあります」と先に出すと強い：

1. **割安の正体は『相場より下振れ』**。値上がり局面で平均ほど上がらなかった駅を拾っている＝好機か構造的衰退かはデータ単独で判別できない。だから Step 12 の検証（玉造=新しい在庫で本物／平野=老朽周縁で要注意）が判別の要。← 最重要。聞かれる前に言う。
2. **対象は20-60㎡帯のみ**（市場の32%）。「大阪市の中古マンション全部」ではなく投資用区分帯。範囲を盛らない。
3. **スコア100点中の実働は約65点**。割安45＋信頼25が主力、地価15は割安と無相関の別軸、品質15・リスク20はほぼ定数/低発火。「配点はv9手順書準拠だが、実測で効いている軸を解剖して把握済み」。
4. **改装96%不明**が1か所でなく3か所（モデル特徴・品質・リスク）を同時に弱める。「効かない特徴をモデルから外すこともできたが、検証済みパイプラインを再カスケードで揺らすリスクが効果ゼロに見合わないので、残して明記する判断をした」＝エンジニアリングの分別を示せる。

清掃の実例：恒久0だった `outlier_pps_flag`（安値<5万専用で価格下限と冗長）をリスクから除去。再ビルドで**スコア完全不変**を確認し「飾りを実態に合わせた」。
