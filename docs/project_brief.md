## 1. ひとことで言うと

**大阪市の中古マンション約7,000件から、「相場より割安に見える＝人が精査すべき物件・駅」を上位に絞り込む、仕入れ一次スクリーニングの道具。**

「買うべき物件」を断定する査定ツールではない。**人間が次に動くための候補を絞る道具**である。

## 2. 誰のどんな課題を解くのか（ビジネス課題）

中古マンションの仕入れ担当は、大量の売り出し物件から「調べる価値のあるもの」を見つけるのに時間を取られる。全件を人手で精査するのは非現実的。

→ **一次スクリーニングを自動化し、精査対象を上位数%に圧縮すれば、仕入れ初期の工数を大きく減らせる。**

これは GA technologies（RENOSY）の「データで割安な投資用中古マンションを仕入れる」事業そのもののミニチュア。本プロジェクトは、その仕入れワークフローの最初の一歩を公開データで再現したもの。

## 3. このツールの立て付け（最重要・割り切り）

| 項目 | 立て付け |
|---|---|
| 目的 | 適正価格の**査定ではない**。**割安候補の検出**（一次フィルター） |
| 評価軸 | 絶対精度ではなく、**駅内での相対順位の頑健さ** |
| 出口 | モデルの出力で終わらせない。**人が次に動ける候補リスト**にする |
| 最終判断 | モデル単独で「買い」と言わない。**必ず人が現地・現物を確認する前提** |

### なぜ精度を追わないのか（実証済みの理由）
- 公開データには値段を左右する情報が無い（階数・部屋番号・正確な住所・リフォーム状態・眺望、面積は5㎡丸め）。
- このため予測のズレには**データ由来の下限**がある。実際、複雑な木モデル(Model E)でも精度は伸びず＝伸びしろはモデルでなくデータ側、と実測で確認。
- よって**モデルの精度追求は費用対効果が悪い**。残り工数は「使える成果物」に全振りする。

### 精度の正直な現状（隠さない）
- 採用モデル(Model C)の中央ズレ約18%、±10%以内は約27%（基準の駅平均17.6%からは改善）。
- **だからこそ「査定」と名乗らず「スクリーナー」と位置づける。** 限界を数字で説明できること自体を価値とする。

## 4. 設計の背骨になっている判断（面接で語れる芯）

- **D-022: アグリゲート最良の Model F をあえて捨て、Model C を採用。**
  Fは全市一律の値上がり補正で集計精度は最高だが、郊外の割安区を構造的に過大予測し「割安候補」を誤検出する。**一様なズレは順位を保つが、構造化したズレは順位＝道具の核心を壊す。** 「点数が高い＝目的に合う、ではない」を実データで示した。
- **D-023: 割安は「駅相場に対する相対」で測る。** 2025の候補同士を比べる限り、一様な時点・駅固定の空間バイアスは駅内比較で相殺される。スクリーニング用途に最も頑健。
- リーク防止（時系列分割・駅統計は訓練期間限定）、欠損補完の理由付け、外れ値の扱い——すべて decision_log に判断理由を記録。

## 4.5 データ用途の限界と成果物の軸（D-024・最重要の割り切り）

使うのは **成約価格情報＝すでに売れ終わった取引**であり、売出中のリストではない。
→ **個別の2025年成約物件を「割安だから買え」とは言えない（もう売れている）。** さらに、相場より安い物件は事故物件・再建築不可・1階北向き・管理不良など**データに写らない理由**で安いこと（逆選択）が多く、物件単位の乖離トップは構造的に地雷へ偏る。

**だから本命の成果物は「物件」ではなく「駅・エリア」とする（D-024）**:
- **駅・エリア単位の割安傾向＝仕入れ部隊がハントすべき優先エリア**。成約データで正当に出せて実務でアクショナブル（どこを探すか決められる）。個別の地雷は集約で薄まる。
- **物件単位は「売出データがあればそのまま本番運用できる手法のデモ＋駅内の乖離分布の可視化」**と位置づける。額面でランキングを信じない。必ず人が現地・現物・謄本を確認する前提。
- 面接の核：「成約データと売出データの違いを理解し、データが正当に支えられる範囲に成果物を絞った。御社の売出・反響データがあれば同じパイプラインがライブのリスティングに対して動く」。

## 5. 成果物の定義（これを完成させる）

1. **駅・エリア別 割安傾向ランキング（本命）** — 仕入れがハントすべき優先エリアを提示。← 道具の本体。最低件数しきい値＋市場信頼性で薄い駅を抑える。
2. **物件単位の乖離分布（補助）** — 駅内で物件がどう散らばるかを可視化。手法デモ。額面ランキングにはしない。
3. **目視検証メモ** — 上位エリア・代表物件が本物っぽいか、なぜ安く出たかを人の目で言語化（逆選択の確認）。
4. **Tableau ダッシュボード** — 地図で駅別割安度→駅を選ぶと物件分布、の2階構造。
5. **物語（README / 説明資料）** — 課題→データ→方法→限界（成約≠売出・逆選択）→使い方→自社データなら何ができるか。

## 6. ビジネスインパクト（「で、いくら得するの?」への答え）

> 大阪市の中古マンション約7,000件から、人手で精査すべき候補を**上位〇〇件（〇%）に圧縮**し、仕入れ初期スクリーニングの工数を概算〇割削減。
> （具体数値は Step 11 の候補リスト確定後に埋める）

## 7. データの限界と「自社データなら何ができるか」（GAに刺さる一文）

現状のハズれの大半は公開データの粒度の壁（階数・正確な住所・リフォーム状態・成約履歴の欠落）に由来する。
**RENOSY が持つような成約履歴・物件詳細・反響データがあれば、この壁を越えて一次スクリーニングを査定精度に近づけられる。** ＝「うちのデータで伸びる人材」だと示す。
