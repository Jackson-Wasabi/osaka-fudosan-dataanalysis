# 大阪市の中古マンション市場分析 ― 仕入れで「次に調べる駅」をデータから絞り込む

### ポートフォリオの目的

本ポートフォリオでは、大阪市の中古マンション市場を対象に、仕入れ・投資の調査を優先すべき駅をデータから絞り込む仕組みを構築しました。

中古マンションの仕入れでは、物件を探す前に「どの駅・エリアから調査するか」を決める必要があります。しかし、大阪市内の多数の駅について、価格相場や取引状況、建物リスクをすべて人手で比較するには大きな手間がかかります。また、単純に価格の安さだけで判断すると、旧耐震物件などのリスクを含む「安い理由がある候補」を選んでしまう可能性があります。

そこで、公開されている成約価格・駅・地価データを用いて、物件ごとの予測㎡単価を算出し、実際の成約価格との乖離から駅ごとの割安傾向を分析しました。さらに、取引件数、価格の安定性、地価、データ品質、建物リスクを組み合わせた**「調査優先度スコア」**を作成し、駅を順位づけしています。

その結果、比較可能な **151駅** から、詳細調査を優先する **10駅** まで候補を絞り込みました。

配点パターンを切り替えると順位（＝仕入れ調査優先駅）が変わる。**堺筋本町・玉造・大国町はどのパターンでも上位＝頑健な本命**。割安重視では伝法・森ノ宮、リスク重視では谷町四丁目が浮上する（配点の意図に応じて狙いが動く）。

## <a id="decisions"></a>2. 設計で大事にした4つの判断

| 問い | どう考えて、どうしたか |
|---|---|
| **Q1. なぜ一番安い物件を本命にしない？** | 安い物件ほど旧耐震×改装不明（訳あり）が多い＝**逆選択**。額面の安さで選ぶと地雷を掴むので、建物リスクを可視化して本命から外した（例：平野）。 |
| **Q2. なぜ一番"当たる"モデルを使わない？** | 精度1位のFは郊外を過大予測して「**偽の割安**」を作り順位を壊す。私の目的は価格当てでなく**駅の順位づけ**なので、Fは見送り、2位でも順位が安定するCを選んだ。 |
| **Q3. この結論はどこまで信じていい？** | 全数値は再現可能。ただし**成約≠売出**なので「買うべき」とは断定せず、最終判断は現物・謄本の確認前提に留めた。 |
| **Q4. なぜ少数データの激安駅を上位にしない？** | **まぐれ安値**を上位に出したくないので、取引10件未満は外し、件数を信頼性スコアに加点した。 |

## <a id="flow"></a>4. 分析フロー

```
不動産情報ライブラリ / 国土数値情報（公開データ）
        │  Python（取得）
        ▼
   BigQuery raw（成約・駅マスタ・地価 を as-is で投入）
        │  dbt
        ▼
   staging（型変換・表記ゆれ補正）  ── stg_transactions / stg_station_master / stg_land_price
        ▼
   intermediate（駅特徴量・地価特徴量・結合・乖離）
        ── int_station_geo / int_station_market_features / int_station_land_price_features
           int_transactions_with_station_features / int_property_deviation
        ▼
   marts（完成テーブル）  ── mart_condo_price（予測用）/ mart_opportunity_list（駅ランキング）
        │  BigQuery ML
        ▼
   予測㎡単価（Model C）→ 駅相対乖離 → 調査優先度スコア
        ▼
   Tableau（3ダッシュボード：サマリー / モデル選定と検証 / 調査優先度）
```

**学習＝2021〜2024年／検証＝2025年**。未来データで検証する時系列分割にして、情報漏洩（リーク）が起きないようにした。

## <a id="preprocessing"></a>5. データ前処理 ― 外れ値・欠損をどう判断したか

ここは一番こだわった工程。ルールで一律に切るのではなく、**列ごとに「なぜそう処理するか」を決めてから**手を動かした（全工程の詳細は [docs/eda_report.md](docs/eda_report.md)）。

**欠損データ：「埋める」か「不明のまま残す」かを、列ごとに使い分け**

![欠損率](outputs/figures/08_missing_rates.png)

- **改装の有無は95.7%が空欄**（大阪市の元データの特性）。ここを無理に推測で埋めると、ありもしない情報を作ってしまう。だから**埋めずに「不明」という印を付け**、その駅のデータの信頼性やリスクを測る材料に使った。
- **建築年（1.9%）・駅からの徒歩分（2.8%）はごく一部だけ欠損**。こちらは**周辺の標準的な値（中央値）で補い**、「補完した」という印を残した。※このとき未来の取引データは使わず、過去のデータだけで埋めている（“答えを先に見てしまう”状態を防ぐため）。
- ほかにも「徒歩30分〜60分」のような曖昧な表記（47件）や、まったく同じ行の重複（330件）も見つけ、数値に直す・重複の印を付ける、で対処した。

**外れ値（極端な値）：数式で機械的に消さず、中身を見て判断**

![築年帯別の㎡単価IQR](outputs/figures/04_box_pps_by_age.png)

- 統計的に「外れ値」と判定された585件も、**すぐには消さなかった**。まず価格のばらつきを扱いやすくする変換をかけ、そのうえで**物理的にあり得ない値だけを除外**。極端な1件1件は実際にデータを目で見て、「事故物件や再建築不可なのか、それとも入力ミスなのか」を見極めた。
- そもそも分析の対象を**面積20〜60㎡・徒歩20分以内・築5〜60年・価格500万円以上**に絞ったので、対象外の極端な物件は自然と外れる形にした。
- 一方で正直な弱点も残る。安すぎる物件用のフィルタは価格の下限設定と重複して意味がなかったので**外した**が、**高すぎる側のチェックは未対応**であることを隠さず明記し、次の改善課題とした。

## <a id="evaluation"></a>6. モデル評価（テスト＝2025年・n=2,097）

| モデル | MAE（平均誤差・円/㎡） | hit10（±10%以内の的中率） | 採用 |
|---|---|---|---|
| Baseline（駅中央値そのまま） | 186,117 | 0.176 | |
| Model A（面積+築年+徒歩） | 135,409 | 0.267 | |
| Model B（A+改装+新耐震） | 135,611 | 0.269 | |
| **Model C（B+駅中央値）** | **120,567** | **0.269** | ✅ **採用** |
| Model F（C+時点index） | 101,093 | 0.329 | ✗（Q2の理由で不採用） |

※ **MAE**＝予測した㎡単価が実際の価格から平均で何円ズレたか（小さいほど高精度）。**hit10**＝予測が実勢価格の ±10% 以内に収まった取引の割合（大きいほど良い）。

**この表は"精度競争"ではない。** hit10（±10%以内）は0.27前後と、点予測としては高くない。だが目的は物件価格を当てることではなく「**どの駅から調べるか**」の順位づけなので、評価の軸も的中率そのものより**駅ランキングが揺れないこと**に置いている。その順位の妥当性は ―― **① 期間をずらした再検証でも hit10 が崩れない**（下記）、**② 配点を3パターン振っても上位駅が安定**（§7 感度分析）―― の2点で確認した。精度1位の F を採らないのも、その精度が郊外の過大予測＝"偽の割安"を生み**順位を壊す**ためだ（§2 Q2）。

選んだCは Baseline 比 **MAE 約35%改善**。駅ごとの相場（駅中央値㎡単価）は、未来の情報が混ざらないよう**その検証で使う"学習期間"のデータだけ**から計算している（過去を学んで未来を当てる設定なので、未来の答えを先取りする「リーク」を防ぐため）。

※ **このMAEは大きい？小さい？** 12万円/㎡のズレは、実勢㎡単価の中央値（約50万円/㎡・大阪市）に対して**約2.4割**。誤差率の中央値でみる **MdAPE は0.183＝予測の半分は実勢の ±約18% 以内**に収まり、Baseline（0.286）から改善した。中古マンションは個別性が大きく使える特徴量も限られるため、点予測の精度はこの水準が現実的。だからこそ"点当て"ではなく駅の順位づけに使っている。

![Baseline vs 採用モデルCの予測精度（±10%・±20%以内の的中率／MdAPE）](outputs/figures/09_model_accuracy.png)

また公示地価（国が出す土地の基準価格）は、**あえて予測モデルには入れず**、スコア側（地価補足15点）でだけ使った。地価を予測に入れると「地価が高い駅だから価格も高い」と学習してしまい、本来見つけたい**“相場より割安”というシグナルを地価で説明し尽くしてしまう**ためだ。→ 詳しい配点は「[調査優先度スコア](#score)」にまとめている。

> ※ 2025年の全対象取引は **2,106件**、評価の **n=2,097 は予測特徴量が揃う行**（差の9件は特徴量欠損で評価対象外）。

- **2025年だけに頼らない（時期をずらして再検証）**：学習と検証の期間の区切りをずらしてもう一度試し（2021〜23年で学習 → 2024年で答え合わせ）、的中率 hit10 が 0.27→0.32 と大きく崩れないことを確認した（たまたま2025年だけ当たった、ではない）。
- **何で効いているかも確認**：標準化係数（ML.WEIGHTS）を見ると効くのは**築年（-0.32で支配的）と駅相場**で、**改装はほぼ0＝価格に効かない**とデータで裏取りした（`sql/bqml/04_inspect_weights.sql`）。

## <a id="score"></a>7. 調査優先度スコア

```
調査優先度 = 割安度45 + 市場信頼性25 + 地価補足15 + データ品質15 − リスク減点20  （0〜100にクリップ）
```

配点に絶対の正解はないので、3パターン（バランス／割安重視／リスク重視）振っても上位に残る駅を **安定候補**、その中で建物リスクが低い駅を **本命** とした（感度分析）。

▶ **配点の詳細（5軸の作り方・3パターンの重み・設計判断）：[docs/scoring_methodology.md](docs/scoring_methodology.md)**

※ 割安度の元になる「駅相対乖離」は、市場全体と駅内で**二段に正規化**して算出（一様な時点要因と駅ごとの空間要因を相殺し、その駅の相場に対する割安度を測る）。

## <a id="verification"></a>8. 結果を疑う ― 上位駅の自己検証

ランキングを出して終わりにせず、**自分が出した上位10駅を「逆選択の指紋」（旧耐震率・大型物件への偏り・薄商い・極端な外れ値の集中）で再検証**した。結果は **本物っぽい割安が4駅／逆選択が濃厚な1駅（平野）／要・現物確認が5駅**。「出した結果を自分で疑う」ところまでを一次スクリーニングの責任範囲とした（詳細：[docs/verification_notes.md](docs/verification_notes.md)）。

## <a id="limitations"></a>11. 分析の限界

- **成約データであり売出ではない**。最終判断は現物・謄本・管理状態・権利関係の確認が前提。
- 対象は中古マンション・面積20〜60㎡・徒歩20分・築5〜60年に限定（市場の約32%）。全市場ではない。
- **改装情報は9割超が欠損**（大阪市で95.7%）。「不明」をリスクとして扱った（逆選択判定の前提）。
- 「割安＝相場より下振れ」は**好機か衰退かを単独では判別できない**。上位駅の逆選択検証（[docs/verification_notes.md](docs/verification_notes.md)）と現物確認が前提。

本プロジェクトの実装にはAIコーディング支援（Claude Code）を活用している。一方で、分析設計の中核 ―― **分析対象の絞り込み、予測特徴量とスコア項目の分離、モデルの採用判断（Model C vs F）、調査優先度スコアの配点、公開データの限界の線引き** ―― はすべて自分で判断し、その理由を [docs/decision_log.md](docs/decision_log.md) に記録した。コードを書く速度をAIで上げつつ、「何を・なぜそう判断したか」を自分の言葉で説明できる状態を重視した。
