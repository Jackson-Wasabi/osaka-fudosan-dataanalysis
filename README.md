# 大阪市 中古マンション ― 仕入れで「次に調べる駅」をデータで絞り込む

大阪市の中古マンション成約データに駅情報と公示地価を組み合わせ、**予測㎡単価と実取引㎡単価の乖離**から「相場より割安に見える＝追加調査すべき駅」を抽出する一次スクリーニング分析です。市場の信頼性・地価・データ品質・建物リスクを加味した**調査優先度スコア**を作り、Tableauで可視化しています。

> **想定ユーザー：** 中古マンションの買取再販／投資の**仕入れ担当**。「どの物件を買うか」ではなく、その手前の「**限られた時間でどの駅から調べるか**」を決める初期スクリーニングを支援します。
>
> 本分析は公開データに基づく一次スクリーニングであり、**「買うべき物件」を断定するものではありません**。実際の仕入れ・投資判断には物件状態・権利関係・管理状況・周辺環境などの追加調査（現物・謄本確認）が必要です。

## 1. 成果（何を解決したか）

人手で当たれば151駅ぶんの相場調査が必要なところを、**調査すべき10駅まで自動で絞り込み**ます。

```
2,106 件          151 駅            66 駅             本命 10 駅
2025年の対象取引 → 相場比較が可能 → 10件以上で安定比較 → 調査対象を約1/14に圧縮
```

| 配点パターン | 仕入れ調査優先 Top5（本命★） |
|---|---|
| 割安重視型(A) | 玉造・大国町・堺筋本町・伝法・森ノ宮 |

※ スコア最上位は平野（割安重視85.6点）だが、**旧耐震×改装不明が86%**＝典型的な「安いには訳がある」逆選択のため本命から自動除外。詳細は §5。

## 4. 分析フロー

```
不動産情報ライブラリ / 国土数値情報（公開データ）
        │  Python（取得）
        ▼
   BigQuery raw（成約・駅マスタ・地価 を as-is で投入）
        │  dbt
        ▼
   staging（型変換・表記ゆれ補正）  ── stg_transactions / stg_station_master / stg_land_price
        ▼
   intermediate（駅特徴量・地価特徴量・結合・乖離）
        ── int_station_geo / int_station_market_features / int_station_land_price_features
           int_transactions_with_station_features / int_property_deviation
        ▼
   marts（完成テーブル）  ── mart_condo_price（予測用）/ mart_opportunity_list（駅ランキング）
        │  BigQuery ML
        ▼
   予測㎡単価（Model C）→ 駅相対乖離 → 調査優先度スコア
        ▼
   Tableau（3ダッシュボード：サマリー / モデル選定と検証 / 調査優先度）
```

**学習＝2021〜2024年／検証＝2025年**。未来データで検証する時系列分割で、情報漏洩（リーク）を排しています。

## 5. 設計で大事にした4つの判断

| 問い | 判断 |
|---|---|
| **Q1. なぜ一番安い物件を本命にしない？** | 安い物件ほど旧耐震×改装不明（訳あり）が多い＝**逆選択**。建物リスクを可視化して本命から自動除外（例：平野）。 |
| **Q2. なぜ一番当たるモデルを使わない？** | 精度1位の Model F（時点index入り）は郊外を過大予測し「**偽の割安**」を作り駅の順位を歪める。目的は予測精度でなく**駅の順位付け**なので、順位が安定する Model C を採用。 |
| **Q3. この結論はどこまで信じていい？** | 全数値が再現可能。ただし**成約≠売出**で、最終判断は現物・謄本確認が必要。 |
| **Q4. なぜ少数データの激安駅を上位にしない？** | 取引10件未満は除外し、件数を信頼性スコアに加点。**少数データのまぐれ安値**を抑える。 |

## 6. モデル評価（テスト＝2025年・n=2,097）

| モデル | MAE（円/㎡） | hit10（±10%以内） | 採用 |
|---|---|---|---|
| Baseline（駅中央値そのまま） | 186,117 | 0.176 | |
| Model A（面積+築年+徒歩） | 135,409 | 0.267 | |
| Model B（A+改装+新耐震） | 135,611 | 0.269 | |
| **Model C（B+駅中央値）** | **120,567** | **0.269** | ✅ **採用** |
| Model F（C+時点index） | 101,093 | 0.329 | ✗（Q2の理由で不採用） |

採用モデルで Baseline 比 **MAE 約35%改善**。駅中央値㎡単価は各Foldの**訓練期間のみ**から算出（リーク防止）。公示地価は予測モデルに入れず、スコア側（地価補足15点）でのみ使用。

### 調査優先度スコア

```
調査優先度 = 割安度45 + 市場信頼性25 + 地価補足15 + データ品質15 − リスク減点20  （0〜100にクリップ）
```

配点を変えた3パターン（バランス／割安重視／リスク重視）でも上位に残る駅を **安定候補** とし、その中で建物リスクが低い駅を **本命** としています（感度分析）。

## 7. 再現手順

```bash
# 1. データ取得（reinfolib 公開フロントエンドキーを使用。環境変数で上書き可）
#    REINFOLIB_KEY=<key> python scripts/download_transactions.py
# 2. BigQuery に raw 投入（成約・駅マスタ・地価）
# 3. dbt build（staging → intermediate → marts）
dbt run && dbt test
# 4. BQML：7モデル作成 → 評価 → Model C 採用
#    sql/bqml/01_create_models.sql → 02_evaluate.sql
# 5. 予測生成 → 駅相対乖離 → 調査優先度スコア（mart_opportunity_list）
#    sql/bqml/03_predict.sql
# 6. Tableau で mart_condo_price / mart_opportunity_list を接続
```

> dbt接続には `~/.dbt/profiles.yml` が必要です。雛形は [profiles.example.yml](profiles.example.yml)。**認証情報・生データはリポジトリに含めていません**（`.gitignore` で除外）。

## 9. 分析の限界

- **成約データであり売出ではない**。最終判断は現物・謄本・管理状態・権利関係の確認が前提。
- 対象は中古マンション・面積20〜60㎡・徒歩20分・築5〜60年に限定（市場の約32%）。全市場ではない。
- **改装情報は約94%が欠損**。「不明」をリスクとして扱った（逆選択判定の前提）。
- 「割安＝相場より下振れ」は**好機か衰退かを単独では判別できない**。上位駅の逆選択検証（[docs/verification_notes.md](docs/verification_notes.md)）と現物確認が前提。
